💡 技术解析

JSON 对象 / 字符串：每次 postMessage 调用时，浏览器会在内部执行结构化克隆算法（Structured Clone Algorithm），这会在主线程与 Worker 之间深度拷贝数据，耗费 CPU 运算资源和内存分配时间。
ArrayBuffer (Transferable)：利用可转移对象（Transferable Objects），在传输时直接将内存的所有权由主线程转移给 Worker（反之亦然）。此操作为零拷贝，耗时极低且不随数据增大而显著增长。
SharedArrayBuffer：多线程直接共享一块连续的物理内存。无需拷贝，也无需转移所有权，配合 Atomics 可以实现几乎瞬时的并发数据存取。
