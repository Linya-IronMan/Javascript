# Promise 异步并发限制器

## 项目说明

实现一个 Promise 任务调度器，用于限制同时执行的异步任务数量。

## 涉及知识点

- Promise 任务调度
- 并发数量控制
- 队列先进先出
- 异步任务完成后的续调度
- 挂起 Promise 与外部 resolve

## 核心思路

调度器内部维护两个关键状态：

- 当前正在执行的任务数量
- 等待执行的任务队列

添加任务时，如果当前并发数量没有达到上限，则立即执行任务；否则将任务放入队列。每个任务完成后，再从队列中取出下一个任务继续执行。

## 总结

该实验适合理解实际业务中的异步并发控制场景，例如批量请求、文件上传、任务池和爬虫限速。
