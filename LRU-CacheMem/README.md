# LRU Cache

## 项目说明

实现一个最近最少使用缓存。缓存容量有限，当容量超出限制时，淘汰最久未被访问的数据。

## 涉及知识点

- LRU 淘汰策略
- `Map` 插入顺序
- 缓存读写
- 数据结构设计
- Jest 单元测试

## 文件说明

| 文件 | 说明 |
|---|---|
| `LRU.ts` | 基于 `Map` 的 LRU 实现 |
| `_LRU.ts` | 另一版实现记录 |
| `LRU.test.ts` | 单元测试 |

## 快速启动

```bash
npm run demo -- lru
```

也可以使用编号启动：

```bash
npm run demo -- 26
```

## 核心思路

`Map` 会按照插入顺序保存键值。访问某个 key 时，先删除再重新插入，就可以把它移动到最新位置。

当缓存数量超过容量限制时，删除 `Map.keys().next().value` 对应的 key，也就是最早插入且最久未被提升的数据。

## 总结

该实验适合理解缓存淘汰策略，以及如何利用语言内置数据结构实现经典算法。
